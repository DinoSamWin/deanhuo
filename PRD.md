## 一、项目概述

### 1.1 项目背景
创建一个展示个人才华和作品的博客网站，集成摄影作品、歌词创作、音乐作品展示以及项目链接。网站采用现代化暗色系设计风格，突出个人品牌形象。

### 1.2 项目目标
- 建立专业的个人品牌展示平台
- 集中展示多类型创作内容（照片、歌词、音乐、项目）
- 提供优质的用户浏览体验
- 实现内容的独立管理和更新

### 1.3 目标用户
- 潜在合作伙伴/客户
- 对创作者作品感兴趣的访客
- 音乐/摄影/技术领域的同行
- 招聘方/猎头

## 二、设计风格参考

### 2.1 整体风格
基于参考图，采用以下设计语言：

| 特征 | 说明 |
|------|------|
| **配色方案** | 深色背景（#0D0D0D ~ #1A1A1A）+ 浅色文字（#FFFFFF）+ 强调色（绿色渐变 #00FF88 ~ #00D4AA） |
| **布局风格** | 卡片式设计（Card-based Layout），模块化布局 |
| **圆角设计** | 大圆角卡片（16px-24px），营造现代柔和感 |
| **字体风格** | 现代无衬线字体，主标题使用手写/艺术字体效果 |
| **交互效果** | 悬停动效、微动画、平滑过渡 |
| **Glassmorphism** | 部分卡片采用毛玻璃效果 |

### 2.2 UI组件规范

#### 颜色系统
```
Primary Background:    #0D0D0D (最深背景)
Secondary Background:  #1A1A1A (卡片背景)
Tertiary Background:   #2A2A2A (次级卡片/悬停)
Primary Text:          #FFFFFF
Secondary Text:        #A0A0A0
Accent Color:          #00FF88 (绿色强调)
Accent Gradient:       linear-gradient(135deg, #00FF88, #00D4AA)
Status Available:      #00FF88 (绿色状态点)
```

#### 字体规范
```
主标题字体:    'Playfair Display' / 'Italiana' (艺术感)
正文字体:      'Inter' / 'Outfit'
代码字体:      'JetBrains Mono'
标题大小:      48px - 64px
副标题:        24px - 32px
正文:          14px - 16px
标签/辅助:     12px
```

## 三、功能模块详述

### 3.1 首页 (Home Page)

首页是网站的核心入口，包含以下区块：

#### 3.1.1 顶部导航栏 (Navigation Bar)
| 元素 | 说明 |
|------|------|
| **位置** | 页面顶部固定 |
| **导航项** | 首页 / 照片 / 歌词 / 音乐 / 项目 |
| **图标风格** | 线性图标（参考图使用：🏠 👤 📁 ➕ 🌐） |
| **CTA按钮** | "Hire me" / "联系我" 按钮（右侧） |
| **交互行为** | **点击导航项在新标签页打开对应详情页** |

#### 3.1.2 个人名片区 (Hero Section / Profile Card)
| 元素 | 说明 |
|------|------|
| **状态标签** | 绿色圆点 + "Available to Hire" 状态 |
| **个人头衔** | 艺术效果的姓名展示 "It's **Dean**" |
| **个人简介** | 2-3行简短介绍（技能、专长、价值主张） |
| **CTA按钮** | "Hire me" / "联系我" |
| **个人照片** | 圆形头像卡片（右侧），带边框效果 |
| **快捷操作** | "Read more about me" / "Copy Email" / "Make schedule" |
| **社交媒体** | 社交图标（微信、B站、抖音、Instagram、等） |

#### 3.1.3 照片推荐模块 (Photos Showcase)
| 元素 | 说明 |
|------|------|
| **标题** | "📸 Recent Photos" + "view all >" 链接 |
| **布局** | 3-4张精选照片瀑布流/网格展示 |
| **卡片效果** | 悬停放大、阴影加深 |
| **交互** | **点击"view all"或单张照片，在新标签页打开照片详情页** |

#### 3.1.4 歌词推荐模块 (Lyrics Showcase)
| 元素 | 说明 |
|------|------|
| **标题** | "📝 Lyrics Collection" + "view all >" 链接 |
| **布局** | 列表/卡片形式展示最新3-5首歌词 |
| **卡片内容** | 歌词标题、创作日期、歌词片段预览 |
| **交互** | **点击卡片或"view all"，在新标签页打开歌词详情页** |

#### 3.1.5 音乐推荐模块 (Music Showcase)
| 元素 | 说明 |
|------|------|
| **标题** | "🎵 Music Works" + "view all >" 链接 |
| **布局** | 横向滑动卡片 / 唱片封面网格 |
| **卡片内容** | 封面图、曲名、时长、播放按钮 |
| **交互** | **点击卡片在新标签页打开音乐详情页，支持快速预览播放** |

#### 3.1.6 项目推荐模块 (Projects Showcase)
| 元素 | 说明 |
|------|------|
| **标题** | "💼 Recent Projects" + "view all >" 链接 |
| **布局** | 大卡片展示项目截图（参考图样式） |
| **卡片内容** | 项目截图、项目名称、技术栈标签、简介 |
| **交互** | **点击卡片在新标签页打开项目链接（外部项目地址）** |

#### 3.1.7 故事/博客模块 (Stories / Explore Thoughts)
| 元素 | 说明 |
|------|------|
| **标题** | "📖 Stories - Explore thoughts" + "view all >" |
| **布局** | 时间线列表形式 |
| **内容** | 作者头像 + 标题预览 + 日期 |
| **交互** | **点击在新标签页打开完整故事/博客** |

---

### 3.2 照片详情页 (Photos Page)

| 功能 | 说明 |
|------|------|
| **页面布局** | 瀑布流画廊 / Masonry Grid |
| **分类筛选** | 按拍摄类型（风景、人像、街拍等）筛选 |
| **照片卡片** | 悬停显示拍摄信息（时间、地点、设备） |
| **灯箱预览** | 点击照片全屏预览，支持左右切换 |
| **照片详情** | 显示EXIF信息（可选） |

---

### 3.3 歌词详情页 (Lyrics Page)

| 功能 | 说明 |
|------|------|
| **页面布局** | 列表页 + 单篇详情页 |
| **列表页** | 卡片列表，展示歌词标题、创作日期、封面（可选） |
| **详情页** | 居中排版的歌词内容，支持滚动动画 |
| **附加功能** | 关联音乐（如有）、歌词创作故事/背景 |

---

### 3.4 音乐详情页 (Music Page)

| 功能 | 说明 |
|------|------|
| **页面布局** | 专辑/单曲网格 |
| **播放器** | 底部固定播放器，支持播放控制 |
| **音乐卡片** | 封面、曲名、时长、播放/暂停按钮 |
| **详情页** | 单曲详情页展示完整信息、歌词、创作背景 |
| **外链支持** | 链接至网易云音乐、QQ音乐、Spotify等平台 |

---

### 3.5 项目详情页 (Projects Page)

| 功能 | 说明 |
|------|------|
| **页面布局** | 项目卡片网格 |
| **卡片内容** | 项目封面图、名称、简介、技术栈标签 |
| **交互行为** | 点击项目卡片跳转至项目外部链接（新标签页） |

## 五、交互规范

### 5.1 导航交互

| 操作 | 行为 |
|------|------|
| 点击导航栏项目 | 在 **新标签页** 打开对应页面 |
| 点击模块"view all" | 在 **新标签页** 打开对应列表页 |
| 点击模块内卡片 | 在 **新标签页** 打开详情页或外链 |
| Logo点击 | 返回首页（当前标签） |

### 5.2 动效规范

| 元素 | 动效 |
|------|------|
| 按钮悬停 | 轻微上浮 + 阴影加深（0.2s ease） |
| 卡片悬停 | 缩放1.02 + 边框高亮（0.3s ease） |
| 页面切换 | 淡入效果（0.3s） |
| 滚动加载 | 元素依次淡入上移（stagger animation） |
| 状态点 | 呼吸灯效果（脉冲动画） |

## 六、技术实现建议

### 6.1 技术栈
| 层级 | 推荐方案 |
|------|----------|
| **前端框架** | 纯 HTML/CSS/JS 或 Vite + Vue/React |
| **样式** | Vanilla CSS（自定义设计系统） |
| **字体** | Google Fonts（Inter, Playfair Display） |
| **图标** | Lucide Icons / Phosphor Icons |
| **动效** | CSS Animations / GSAP（可选） |
| **图片** | 懒加载 + WebP格式优化 |
| **音乐播放** | HTML5 Audio API |

### 6.2 响应式断点
```css
/* Mobile First */
Mobile:    < 768px
Tablet:    768px - 1024px  
Desktop:   1024px - 1440px
Large:     > 1440px
```

### 6.3 SEO优化
- 语义化HTML结构
- Meta标签完善（title, description, og:image）
- 结构化数据（Schema.org）
- 图片Alt文本
- 页面加载性能优化

## 七、内容管理需求

### 7.1 首页内容
首页内容由站长单独上传管理，包括：
- 个人信息（头像、姓名、简介、社交媒体链接）
- 各模块推荐内容的选择和排序

### 7.2 内容结构
建议采用JSON/Markdown文件管理静态内容：

```
/content
  ├── /photos
  │   ├── photos.json       # 照片元数据
  │   └── /images           # 照片文件
  ├── /lyrics
  │   ├── lyrics.json       # 歌词列表
  │   └── /*.md             # 单篇歌词Markdown
  ├── /music
  │   ├── music.json        # 音乐列表
  │   └── /covers           # 封面图片
  └── /projects
      └── projects.json     # 项目列表
```
